# v5.3.0 — Apple Silicon Emitter Design

**Status:** Phase 1 complete (baseline, audit, design). Phases 2–4 pending.
**Target:** Replace the stub `EMITMACHO_ARM64` with a real emitter that
matches the v5.2.3 probe layout. Enable `cyrius build` to produce arm64
Mach-O binaries that run on Apple Silicon after `codesign -s -`.

## Why a design doc?

The probe work in v5.2.3 (`programs/macho_probe_arm.cyr`,
`programs/macho_probe_arm_hello.cyr`) proved the format. This doc
maps that proof onto the compiler so three future sessions can land
cleanly. Vidya + doc audit = Phase 4 closeout, not Phase 1.

## Baseline (v5.2.3 bench)

`cc5 size = 417744 bytes`. `compiler/self_compile = 244ms`.
`compiler/trivial = 2ms`. Re-run `cyrius bench` after Phase 3 to
confirm the emitter adds no more than ~5% overhead on self_compile.

## Current state

### What works today
- x86_64 + aarch64 ELF emission (production)
- x86_64 Mach-O emission (v5.1.0, `CYRIUS_MACHO=1`)
- arm64 Mach-O emitter *stub* — `EMITMACHO_ARM64` in
  `src/backend/macho/emit.cyr:185`. Produces LC_UNIXTHREAD + RWX +
  raw-SVC output that is rejected by the macOS kernel on every gate:
  unsigned, non-PIE, RWX, raw syscalls.

### What the v5.2.3 probes proved
Apple Silicon binaries need all of:
1. `MH_PIE` flag
2. `MH_DYLDLINK` + `MH_TWOLEVEL`
3. `LC_MAIN` (not `LC_UNIXTHREAD`)
4. `LC_LOAD_DYLINKER` + `LC_LOAD_DYLIB libSystem.B`
5. `LC_BUILD_VERSION` (platform=macOS, minos≥11)
6. `LC_DYLD_CHAINED_FIXUPS` with a well-formed (possibly empty) blob
7. `__TEXT` segment R|X (not RWX), 16KB pages
8. `__DATA_CONST.__got` with DYLD_CHAINED_PTR_64 bind entries per import
9. `__stubs` section (3 instrs per imported libSystem fn)
10. `__LINKEDIT` with chain fixups + symtab + strtab + exports trie
    + function starts + data in code
11. Ad-hoc `codesign -s - --force` post-emission (user runs this)

### Syscall emission sites (audit)
The Mach-O ARM path must intercept every place that emits `svc #0`:
- `src/backend/aarch64/emit.cyr:141` — `ESYSCALL` (emits `0xD4000001`)
- Callers: `ESCPOPS` (268), `EEXIT` (299), `common/ir.cyr:374` when
  replaying `IR_SYSCALL`
- `src/backend/aarch64/emit.cyr:529` — inline `svc` mnemonic in asm
  block (separate path — do we care? Probably reject in Mach-O ARM mode)

### Parser already captures syscall numbers
`src/frontend/parse.cyr:683` — when the first arg of `syscall(...)`
is a compile-time constant, `_cfo=1` and `_cfv=sc_num`. This is
already used for arity warnings. **We get the syscall number for
free at parse time.** The Mach-O ARM path keys off this.

## Phase 2 — compiler state + emitter skeleton

### Heap map additions
New region at `0xD8000` (currently free; was struct_fnames, moved
to 0xDCA000 in v4.x). 8 KB supports 256 imports × 32 bytes:

```
0xD8000  macho_imports          [8192]   256 imports × 32 B
  Per entry:
    +0   name_ptr    (8)    pointer into str_data (interned)
    +8   stub_off    (4)    byte offset of stub in __stubs section
    +12  got_off     (4)    byte offset of pointer in __got
    +16  sc_num      (4)    Linux x86_64 syscall number (for mapping)
    +20  flags       (4)    bit 0 = registered
    +24  reserved    (8)    future use
0xD9F00  macho_import_count     (8)      count of registered imports
```

Add `macho_import_count` read/write helpers (`GMIC`/`SMIC` style)
matching existing state-accessor conventions.

### Import registration API (in `src/backend/macho/imports.cyr` — NEW)

```
# Idempotent: returns existing index or allocates new.
fn macho_import_register(S, name_cstring) -> int
# Lookup by index: returns 0 if out of range.
fn macho_import_name(S, idx) -> cstring
fn macho_import_count(S) -> int
# Map Linux syscall num → "_write", "_read", etc.
# Returns 0 (null) for syscalls with no macOS analog (e.g. brk).
fn macho_syscall_to_libsystem(sc_num) -> cstring
```

The mapping table (authoritative for v5.3.0):

| Linux x86_64 sc | libSystem fn | Notes |
|---|---|---|
| 0 SYS_READ | `_read` | |
| 1 SYS_WRITE | `_write` | |
| 2 SYS_OPEN | `_open` | |
| 3 SYS_CLOSE | `_close` | |
| 8 SYS_LSEEK | `_lseek` | |
| 9 SYS_MMAP | `_mmap` | |
| 10 SYS_MPROTECT | `_mprotect` | |
| 11 SYS_MUNMAP | `_munmap` | |
| 12 SYS_BRK | — | no macOS analog, compile error |
| 60 SYS_EXIT | `_exit` | rarely needed — LC_MAIN main-return calls exit automatically |
| 228 SYS_CLOCK_GETTIME | `_clock_gettime` | |

### EMITMACHO_ARM64 rewrite (byte-level parity with probe)

Replace the current ~110 LOC stub with a ~400 LOC emitter that emits,
in order, starting from `O = S + 0x64A000`:

1. `mach_header_64` (32 B)
2. `LC_SEGMENT_64 __PAGEZERO` (72 B)
3. `LC_SEGMENT_64 __TEXT` — cmdsize depends on number of
   subsections (at least `__text`, `__stubs`, `__cstring`,
   `__unwind_info` when we have imports). If zero imports,
   `__stubs` is omitted and cmdsize = 72 + 3·80 = 312.
4. `LC_SEGMENT_64 __DATA_CONST` — **only emitted if imports > 0**.
   Contains one `__got` section of `imports·8` bytes.
5. `LC_SEGMENT_64 __LINKEDIT`
6. `LC_DYLD_CHAINED_FIXUPS` — blob size varies with import count.
   Zero imports = 56 B (as in Stage 2). Non-zero = header(32) +
   starts_in_image(seg_count·4+4, padded) + starts_in_segment(24
   for __DATA_CONST page) + imports·4 + symbol strings.
7. `LC_DYLD_EXPORTS_TRIE` — 48 B for `__mh_execute_header` + `_main`
8. `LC_SYMTAB` — nsyms = 2 (fixed) + imports
9. `LC_DYSYMTAB` — nundefsym = imports, nindirectsyms = 2·imports
10. `LC_LOAD_DYLINKER` (32 B)
11. `LC_UUID` (24 B) — deterministic UUID derived from some hash of
    the program content so self-hosting stays byte-identical
12. `LC_BUILD_VERSION` (32 B)
13. `LC_SOURCE_VERSION` (16 B)
14. `LC_MAIN` (24 B) — entryoff = (TEXT_OFFSET + main_code_offset)
15. `LC_LOAD_DYLIB /usr/lib/libSystem.B.dylib` (56 B)
16. `LC_FUNCTION_STARTS` (16 B)
17. `LC_DATA_IN_CODE` (16 B)

### Self-hosting UUID concern

The x86_64 Mach-O emitter doesn't emit UUID (it's pre-Big-Sur and
x86_64 macOS doesn't strictly require it). arm64 does require it.
Clang generates random UUIDs per build, which makes byte-identical
self-hosting verification impossible if the UUID is random.

**Decision:** emit a deterministic UUID by hashing program content
(e.g. FNV-1a over the concatenation of __text + __DATA_CONST +
string data). Self-hosting stays byte-identical because same input
→ same UUID. Document the hash algorithm in the emitter.

## Phase 3 — binding + codegen swap

### Parser hook
At `src/frontend/parse.cyr:691` (before `ESCPOPS(S, argc)`):

```
if (_TARGET_MACHO == 2) {
    if (sc_num < 0) {
        # Dynamic syscall number — unsupported on Mach-O ARM
        ERR(S, "Mach-O ARM requires constant syscall numbers", 46);
    }
    var libfn = macho_syscall_to_libsystem(sc_num);
    if (libfn == 0) {
        ERR(S, "syscall has no macOS libSystem equivalent", 41);
    }
    var imp_idx = macho_import_register(S, libfn);
    ESCPOPS_MACHO(S, argc, imp_idx);
} else {
    ESCPOPS(S, argc);
}
```

### New aarch64 emit function
`src/backend/aarch64/emit.cyr`:

```
# Pop syscall args into x0-x5 (skipping x8 — libSystem calls don't
# use x8), then bl to __stub_N. The branch offset is patched by
# FIXUP once __stubs section placement is known.
fn ESCPOPS_MACHO(S, n, imp_idx) {
    # Pop argN..arg1 into x5..x1 (same as Linux path).
    # DROP the syscall_num pop entirely — libSystem is called by name.
    if (n >= 7) { EPOPR(S); EW(S, 0xAA0003E5); }
    if (n >= 6) { EPOPR(S); EW(S, 0xAA0003E4); }
    if (n >= 5) { EPOPR(S); EW(S, 0xAA0003E3); }
    if (n >= 4) { EPOPR(S); EW(S, 0xAA0003E2); }
    if (n >= 3) { EPOPR(S); EW(S, 0xAA0003E1); }
    if (n >= 2) { EPOPR(S); }  # arg1 -> x0
    if (n >= 1) { EPOPR(S); DROP(S); }  # drop syscall num
    # Emit `bl __stub_{imp_idx}` as a placeholder — FIXUP patches it.
    EW_FIXUP_STUB(S, imp_idx);  # new fixup kind
    return 0;
}
```

### New fixup kind
Extend the fixup table (`0xA0000`) with a kind tagging each entry as
`FIXUP_BL_STUB` — at FIXUP time, the patched address = `__stubs.addr
+ imp_idx * 12`. Since `bl` has a ±128 MB reach and Mach-O binaries
are well under that, a single-instruction branch always suffices.

### Stub section emission
In `EMITMACHO_ARM64`, after emitting main code but before padding
to page, emit one 12-byte stub per imported symbol:

```
for i in 0..imports_count:
    got_vaddr = DATA_CONST_VMADDR + i*8
    adrp_imm = (got_vaddr >> 12) - (stub_vaddr >> 12)
    EW(O, 0x90000010 | encode_adrp_imm(adrp_imm))  # adrp x16, got_page
    EW(O, 0xF9400210 | ((got_vaddr & 0xFFF) >> 3) << 10)  # ldr x16, [x16, #off]
    EW(O, 0xD61F0200)  # br x16
```

### __got + chain fixups per import
`__DATA_CONST.__got` has `imports_count` 8-byte slots, each initialized
to a DYLD_CHAINED_PTR_64 bind entry:
```
value = (1ULL << 63)        # bind bit
      | (ordinal & 0xFFFFFF) # import index into chain_fixups.imports
```

Chain fixups blob construction follows the probe exactly, parameterized
by imports_count. Symbol pool format: single null byte prefix, then
each imported name null-terminated, concatenated.

## Risks

- **UUID determinism** — if the hash input accidentally includes a
  non-deterministic byte (e.g. timestamp from a missed buffer init),
  self-hosting fails. Mitigation: exhaustively zero the emitter's
  working buffer before each run; unit-test the UUID hash.
- **Chain fixup alignment** — spec is strict about 8-byte alignment
  of the imports array and symbols pool. Easy to get wrong. The
  v5.2.3 probe establishes the exact layout; replicate pedantically.
- **LC_MAIN vs dyld runtime** — when `main` returns, dyld's startup
  runtime calls `_exit(rc)`. This means `EEXIT` (the implicit
  compiler-inserted exit at program end) is redundant on Mach-O ARM
  but harmless (it'd be an explicit `syscall(60, 0)` → `bl _exit`).
  Verify no unintended interaction with defer blocks.
- **Bootstrap integrity** — `bridge.cyr` has its own `ESYSCALL` but
  bridge doesn't run on Mach-O ARM (Linux-only). Safe to leave.

## Non-goals for v5.3.0

- x86_64 macOS parity enhancements (that's v5.1.x territory)
- libSystem malloc/free — stdlib `alloc_macos.cyr` already uses
  `_mmap` directly, works fine with the new stub mechanism
- Windows PE/COFF — v5.4.0
- Dynamic syscall numbers — reject at compile time, document as
  target limitation
