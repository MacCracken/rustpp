## §3 — Benchmark baseline (**BLOCKED at audit; UNBLOCKED 2026-05-01 via v5.7.50**)

**RESOLUTION (2026-05-01)**: Closed by **v5.7.50** — single config-
only fix to `cyrius.cyml`. The recommendation in this section's
original write-up (add stdlib wrappers) was reframed during execution:
adding wrappers does NOT immediately fix patra 1.9.2's existing raw
`syscall(SYS_LSEEK, ...)` references — only future patra versions
that migrate to `sys_lseek(...)` would benefit. The actual fix was
adding the `[deps].stdlib = [...]` block that every other consumer
(yukti / patra / sigil / etc.) has had since their fold. Cyrius's
own cyrius.cyml was the last to need one. After v5.7.50: `cyrius
bench` 15/15 PASS, `cyrius test` unblocked. The wrapper API gap
(separate concern) stays as v5.8.x slot candidate alongside phylax #2.

Original analysis (kept for audit-trail clarity):


`cyrius bench` and `cyrius test` both fail at compile-time:

```
error:lib/patra.cyr:101: undefined variable 'SYS_LSEEK' (missing include or enum?)
FAIL: compile error
```

13 of 15 bench files fail with the same error; 2 (mulmod,
miller_rabin) pass because they don't auto-prepend patra.

**Root cause**: `cyrius bench` / `cyrius test` auto-prepend the
`[deps].*` distfiles from `cyrius.cyml`. patra's distfile
(`lib/patra.cyr`, line 101) calls `syscall(SYS_LSEEK, fd, off, 0)`
directly. patra does NOT `include "lib/syscalls.cyr"` in its dist;
cyrius's auto-prepend chain doesn't pre-include `lib/syscalls.cyr`
before patra. So `SYS_LSEEK` is unresolved at the time patra's body
parses.

**Pre-existing**: patra 1.9.0 (the v5.7.48 pin) had identical
`SYS_LSEEK` references at lines 101 / 403 / 2968. The v5.7.49
deps bump (1.9.0 → 1.9.2) did NOT introduce this — 1.9.2's main
change was migrating other syscalls (SYS_OPEN/READ/WRITE/CLOSE) to
`sys_open`/`sys_read`/`sys_write`/`sys_close` wrappers. SYS_LSEEK
stayed raw because there's no `sys_lseek` wrapper in stdlib (same
gap class as phylax #2's missing `sys_stat`/`sys_fstat`).

This issue was latent at v5.7.48 (and earlier) and only surfaced
because no recent closeout exercised `cyrius bench` end-to-end —
check.sh uses cc5 directly without the auto-prepend layer.

**Action for v5.8.0**: file as v5.8.x slot candidate (see audit
findings doc). Three fix shapes worth considering:

1. **Add `sys_lseek` wrapper to stdlib** (and any other missing
   wrappers — `sys_fdatasync`, `sys_flock`, `sys_fsync`). Patra
   (and other deps) call wrappers; cyrius's auto-prepend already
   includes `lib/syscalls.cyr`'s wrapper-side. Pairs naturally
   with phylax #2's `sys_stat`/`sys_fstat` backfill.
2. **Auto-prepend `lib/syscalls.cyr` before any dep that uses
   raw `syscall(SYS_*, ...)`**. More general but requires
   detecting which deps need the dispatcher.
3. **Patra-side**: add `include "lib/syscalls.cyr"` to patra's
   dist. Solves it at source but only for patra; same gap can
   recur in any new dep.

**Recommendation**: do (1) — covers patra's raw uses + addresses
phylax #2 + sets a precedent (deps call wrappers, not raw syscalls).

**Bench baseline cannot be captured until this lands.** v5.8.0 P(-1)
post-audit benchmark step can't compare against pre-audit baseline —
re-baseline once `cyrius bench` runs clean.

## §4 — Audit findings

### Stale code

- **`build/cyrc_check`** (12,344 B, executable, ELF) — orphan
  binary. Zero references in `scripts/`, `.github/`, `tests/`. Same
  size as `build/cyrc`. Likely a leftover from a debug or
  parity-check session. Action: delete or document purpose.

- **Stale version refs in comments** (10+ hits) — historical
  context like `src/main_cx.cyr:9 "v5.7.11: ir.cyr added at
  v5.6.12"`, `src/backend/aarch64/fixup.cyr:9 "v5.6.5 O1 +
  v5.6.6"`. Mostly fine as historical anchoring; trim only those
  that mislead about current state. Low priority.

### Real TODOs

- **`src/backend/cx/emit.cyr:385`** — `ESTORESTACKPARM` stub
  returns 0 with comment `# TODO: >6 args`. cx backend can't
  handle stack-arg passing for fns with >6 formal params. Same
  ABI-leak risk as the pre-v5.7.27 path. **Pin candidate** for
  v5.8.x — file as a slot if a cx consumer surfaces a 7+-arg fn.

### Optimization opportunities

- **`src/backend/x86/float.cyr:41`** — comment describes a
  5-instruction pattern `push rax; movabs rax, 0x7FFF...; mov rcx,
  rax; pop rax; and rax, rcx`. The `push rax` / `pop rax` framing
  the constant load is the v5.6.9 redundant-push/pop class — but
  here the `pop rax` is preceded by `mov rcx, rax`, so the value is
  saved before the pop reuses rax. May still be reducible:
  `movabs rcx, 0x7FFF...; and rax, rcx` (3 bytes saved per site).
  **Pin candidate** for v5.8.x optimization arc — measure first.

- **No heap-cap pressure** — all 80 regions clean, 0 warnings.
  No region near its cap. The v5.7.x cycle (codebuf 1MB → 3MB,
  input_buf 512KB → 1MB, fixup 32K → 1M) absorbed most of the
  caps that mattered. Heap stays as v5.8.0 baseline.
